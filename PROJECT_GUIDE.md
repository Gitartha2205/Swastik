# Project Title: Swastik

## Overview
Swastik is an HR management system designed to streamline various HR functions such as employee management, recruitment, attendance tracking, and performance evaluation. This application aims to minimize administrative tasks, improve communication, and enhance the overall employee experience within organizations.

## Features
- **Employee Management**: Add, update, and manage employee records.
- **Recruitment Module**: Post job openings, manage applications, and track recruitment processes.
- **Attendance System**: Record attendance, manage leave requests, and generate reports.
- **Performance Evaluation**: Conduct performance reviews and manage feedback from different departments.
- **Reporting**: Generate various HR-related reports for analysis.

## Installation
To get started with Swastik, follow these instructions:

1. **Clone the Repository**:
   ```bash
   git clone https://github.com/Gitartha2205/Swastik.git
   cd Swastik
   ```
   
2. **Install Dependencies**:
   ```bash
   npm install
   ```

3. **Set Up the Database**:
   Ensure that you have a database configured. Follow the instructions in the `database_setup.md` for details.

4. **Run the Application**:
   ```bash
   npm start
   ```
   Visit `http://localhost:3000` in your web browser to access the application.

## Demo Guide
To demonstrate the capabilities of Swastik:

1. **Log In**: Use the credentials provided in the setup instructions to log in to the application.
2. **Add an Employee**: Navigate to the Employee Management section and add a new employee.
3. **Track Attendance**: Go to the Attendance section, record attendance for employees, and view the attendance report.
4. **Evaluate Performance**: Under the Performance section, conduct a performance review for an employee and provide feedback.
5. **Generate Reports**: Use the Reporting module to generate and view comprehensive HR reports.

Swastik is designed to enhance the efficiency of HR processes, providing valuable tools to manage human resources effectively. For further information, please refer to the documentation or contact the project maintainers.
